Design a **library system** with the next rules:
- A **Book** has title, author, ISBN.
- An **User** can loan and returb books.
- The system has to register each loan.
- Apply SOLID principles.


SOLID Principles:
- S - Single responsability: Each class has just one task.
- O - Open/Close: Open for the implementations but close for any changes to an existing class.
- L - Liskov Substitution: A child class can replace anything from the father class without breaking anything.
- I -- Interface segregation: Just add the thing you really use to each class.
- D - Dependency Inversion: Use abstaction dependencies instead of using a concrete thing for a class.
